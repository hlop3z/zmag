    Welcome to Zmag

    Zmag, is built with Pyzmq (ZeroMQ) and Strawberry this project is the sibling of Fastberry

    The Command-Line-Interface (CLI) is built with Click.

    Description

    A tool for building GraphQL — API(s) with Python.

    You can create . . .

    1. GraphQL — Query(s) and Mutation(s).
    2. Commands — To create automated processes and more . . .

    Built With

    Module Is Used To...
    Pyzmq API's core Universal Messaging Library.
    Strawberry GraphQL Library
    Click Manage the server, development process and custom Commands.
    SPOC FrameWork tool for building this Framework.
    DBController Database Controller for SQL and Mongo.
    Starlette Run the server in Debug mode.

    Install Zmag (Demo)

    python -m pip install "zmag[testing]"

    Install Zmag Mongo

    python -m pip install "zmag[mongo]"

    Install Zmag SQL

    python -m pip install "zmag[sql]" "databases[sqlite]"

    SQL Options

    Database Extra Installation(s)
    PostgreSQL "databases[postgresql]"
    MySQL "databases[mysql]"
    Sqlite "databases[sqlite]"

    Inspired By Django

    There are several things from Django that inspire this tool.

    Some of the commands and the installation of modules (aka: INSTALLED_APPS) inside a Django project.

    Zmag comes with a few key commands:

    Command Is Used To...
    startproject Create a new Zmag project.
    ./manage.py run Run FastApi Server.
    ./manage.py start-app Create a Zmag App inside your "apps" directory.
    ./manage.py --help For more information.

    startproject

    Careful with the command startproject. Only use it once and make sure you are in a new folder. It will write files and folders.
